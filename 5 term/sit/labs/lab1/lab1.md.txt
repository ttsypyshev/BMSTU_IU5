Приведу подробное руководство для выполнения задания в Cisco Packet Tracer, шаг за шагом. 

### Шаг 1: Добавление устройств
1. **Добавьте два коммутатора (Switch)**:
   - Перейдите в панель инструментов снизу.
   - Выберите вкладку `Switches`, затем перетащите два коммутатора **2960** на рабочую область.
   - Назовите их `Switch1` и `Switch2` (вы можете сделать это, щелкнув правой кнопкой мыши на коммутаторе и выбрав "Настройки").

2. **Добавьте 10 компьютеров**:
   - Выберите вкладку `End Devices` и добавьте 10 компьютеров **(PC)**, разметив их на рабочей области.
   - Назовите 7 компьютеров, подключаемых к `Switch1`, например `PC1` – `PC7`.
   - Оставшиеся 3 компьютера подключите к `Switch2` и назовите их `PC8`, `PC9`, и `PC10`.

### Шаг 2: Подключение устройств кабелями
1. **Подключите компьютеры к коммутаторам**:
   - Используйте **прямой кабель (Copper Straight-Through)** для подключения каждого компьютера к коммутатору.
   - Подключите `PC1` – `PC7` к `Switch1` (к портам **FastEthernet 0/1** – **FastEthernet 0/7**).
   - Подключите `PC8` – `PC10` к `Switch2` (к портам **FastEthernet 0/1** – **FastEthernet 0/3**).

2. **Подключите коммутаторы друг к другу**:
   - Используйте **кроссоверный кабель (Copper Cross-Over)** для соединения `Switch1` и `Switch2`.
   - Подключите кабель к порту **FastEthernet 0/24** на `Switch1` и к порту **FastEthernet 0/24** на `Switch2`.

3. **Настройте консольное подключение для `Switch1`**:
   - Добавьте одно устройство типа **PC** (назовем его `ConsolePC`) рядом с `Switch1`.
   - Используйте **консольный кабель (Console Cable)** для подключения `ConsolePC` к `Switch1`:
     - Подключите кабель к порту **RS 232** на `ConsolePC`.
     - Подключите другой конец к порту **Console** на `Switch1`.

### Шаг 3: Настройка IP-адресов на компьютерах
1. Выберите каждый компьютер и задайте IP-адреса, соответствующие одной подсети.
2. **IP-адреса и маска подсети для компьютеров**:
   - Откройте каждый ПК, выберите `Desktop` → `IP Configuration`.
   - Задайте IP-адреса следующим образом:
     - `PC1`: 192.168.51.1 / 255.255.255.0
     - `PC2`: 192.168.51.2 / 255.255.255.0
     - и так далее, до `PC10`: 192.168.51.10 / 255.255.255.0

### Шаг 4: Конфигурация коммутаторов
Теперь начнем настраивать сами коммутаторы. Начнем с консольного подключения.

#### Конфигурация `Switch1` через `ConsolePC`
1. Выберите `ConsolePC`, перейдите на вкладку `Desktop` → `Terminal`.
2. Откроется окно терминала с настройками по умолчанию, просто нажмите **OK**.
3. Введите команды для настройки `Switch1`:
   ```bash
   enable
   configure terminal
   ```

#### Настройка trunk-соединения и Native VLAN
4. Настройте trunk-соединение между `Switch1` и `Switch2` на портах **FastEthernet 0/24**:
   - На `Switch1`:
     ```bash
     interface FastEthernet 0/24
     switchport mode trunk
     switchport trunk native vlan 99  # Изменим Native VLAN на 99
     exit
     ```
   - На `Switch2`:
     ```bash
     enable
     configure terminal
     interface FastEthernet 0/24
     switchport mode trunk
     switchport trunk native vlan 99
     exit
     ```

5. **Настройка Port Security на одном из интерфейсов**:
   - Вернитесь на `Switch1` и настройте **Port Security** для одного из портов, например, **FastEthernet 0/1**:
     ```bash
     interface FastEthernet 0/1
     switchport mode access
     switchport port-security
     switchport port-security maximum 1
     switchport port-security violation restrict
     exit
     ```

#### Демонстрация Port Security
6. **Для проверки работы Port Security**:
   - Попробуйте добавить еще один ПК и подключить его к `Switch1` на `FastEthernet 0/1`.
   - Вы увидите, что коммутатор не позволяет подключить более одного устройства.

#### Дополнительная настройка: Telnet на `Switch1`
7. **Включение Telnet-доступа к `Switch1`**:
   - На `Switch1` введите:
     ```bash
     line vty 0 4
     password cisco123  # Установите пароль на Telnet-доступ
     login
     exit
     ```
   - Теперь вы сможете подключиться к `Switch1` по Telnet с любого ПК в сети, используя IP-адрес `Switch1`.

### Шаг 5: Проверка связи между устройствами
1. Для проверки связности сети, используйте инструмент **ping**:
   - Выберите любой компьютер и перейдите на `Desktop` → `Command Prompt`.
   - Введите `ping <IP другого компьютера>`, чтобы проверить связь между устройствами.

2. **Проверка trunk соединения и структуры пакетов**:
   - Перейдите в режим симуляции в Packet Tracer.
   - Отправьте пакеты между `Switch1` и `Switch2`, а также между компьютерами. Вы увидите различия в пакетах, так как trunk-соединение использует тегирование VLAN.

Теперь ваша сеть полностью настроена.
