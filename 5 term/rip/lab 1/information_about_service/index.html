Задание:
Требуется разработать дизайн трех страниц приложения в figma, повторив при этом стилистику уже существующего популярного ресурса, связанного с вашей темой. Данные по теме также нужно искать реальные.
Создание базового интерфейса, состоящего из трех страниц. Первая для просмотра списка услуг (отели, товары, рейсы и тд) в виде карточек с наименованием, ценой и картинкой, а также отдельным элементом-карточкой текущей заявки (корзина) с указанием количества услуг в ней. Количество вычисляется. При клике по карточке происходит переход на вторую страницу с подробной информацией об услуге (даты, описание и тд). А при нажатии на текущую заявку по ее id открывается страница просмотра состава заявки. Добавить кнопку/иконку Домой для навигации на страницу услуг.
В приложении должны быть использованы стили, для каждого элемента списка подгружается свое изображение. Изображения хранятся в Minio, наименование изображение совпадает с id услуги, но хранится в модели отдельным полем. Разработать стиль приложения, который будет применяться далее в последующих лабораторных по фронтенду. CSS вынести в отдельный файл. Все данные для обеих страниц нужно брать прямо из двух коллекций (услуг и заявки), без использования БД.
Добавить поле input для фильтрации на сервере списка услуг по одному из полей (наименование, цена), отображаемых на странице (по умолчанию отображать все). Поле поиска должно сохраняться после запроса. Всего в приложении должно быть 3 GET запроса и две модели-коллекции. Никакого редактирования, только поиск и просмотр. Без JavaScript Порядок показа:
показать три страницы приложения, во вкладке Network объяснить заголовки и свой HTML из Response, объяснить шаблоны, контроллеры этих страниц и коллекции данных. Показать Minio, Figma

Контрольные вопросы:
